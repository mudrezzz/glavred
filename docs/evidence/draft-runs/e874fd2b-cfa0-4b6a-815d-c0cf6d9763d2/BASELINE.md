## Как сравнивать следующие запуски

Для нового запуска нужно сохранить новый `runId` и выполнить те же проверки:

```powershell
python .agents/skills/draft-run-pipeline-diagnostics/scripts/analyze_draft_run.py <run-id>
python scripts/analyze_draft_run_reliability.py --run-id <run-id> --format markdown
python scripts/audit_draft_run_provider_inputs.py --run-id <run-id> --format markdown
python scripts/audit_draft_run_provider_dossiers.py --run-id <run-id> --format markdown
```

Сравнивать следует запуски с теми же проектом, темой, фабулой, настройками и
ролями моделей. Текст LLM может различаться случайно, поэтому главным доказательством
улучшения считаются stage-level метрики: размер provider-входов, бюджет,
таймауты, fallback, evidence coverage, открытые critical/warning и финальный
вердикт.
