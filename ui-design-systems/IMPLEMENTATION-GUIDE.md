# IMPLEMENTATION-GUIDE — как собрать фронтенд на дизайн-системе «Главред»

Практическое руководство: как запустить референсы, как встроить токены в реальный
проект и на что обратить внимание при переносе в продакшен. Бренд-правила — в
`design-system.md`; точные значения компонентов — в `COMPONENT-SPECS.md`.

---

## 1. Запуск референсов локально

Все референсы — статические HTML и открываются двойным кликом. Для надёжности
(некоторые браузеры ограничивают локальную загрузку модулей) лучше поднять простой
статический сервер из корня папки:

```bash
# любой из вариантов, из папки glavred-frontend-handoff/
python3 -m http.server 8080
# или
npx serve .
```

Затем открыть:
- `http://localhost:8080/spec-previews/index.html` — галерея токенов и компонентов
- `http://localhost:8080/ui-kit/glavred-app/index.html` — приложение
- `http://localhost:8080/ui-kit/marketing-site/index.html` — лендинг

> Киты тянут React 18 и Babel с CDN — нужен интернет. JSX транспилируется в браузере;
> это сделано ради простоты просмотра и **не предназначено для продакшена**.

---

## 2. Выбор стека (если боевого ещё нет)

Система не привязана к фреймворку — это слой CSS-токенов плюс правила. Рекомендации:

- **React + Vite + TypeScript** — ближе всего к структуре китов, перенос самый прямой.
- **Vue / Svelte / SolidJS** — без проблем; токены и спеки фреймворк-агностичны.
- **Next / Nuxt / Remix** — если нужен SSR/маршрутизация из коробки.

Для стилей подойдёт любой подход — CSS-модули, vanilla CSS, styled-components,
Tailwind (см. §5). Главное условие во всех случаях: **значения берутся из токенов,
а не пишутся руками.**

---

## 3. Подключение токенов

Скопируйте `colors_and_type.css` в проект (например, `src/styles/tokens.css`) и
подключите **первым**, до любых своих стилей и до стилей компонентов.

```html
<!-- vanilla -->
<link rel="stylesheet" href="/styles/tokens.css">
```

```ts
// Vite / любой бандлер — в точке входа
import './styles/tokens.css';
```

Файл делает три вещи:
1. `@import` подтягивает шрифты (Golos Text, Lora, JetBrains Mono) с Google Fonts.
2. Объявляет все токены как CSS-переменные в `:root`.
3. Даёт семантические дефолты для «сырого» HTML через классы-обёртки `.gr-ui` и
   `.gr-editorial` (см. §6).

После этого в любом своём стиле:

```css
.btn-primary {
  font: var(--ui-md);
  font-weight: 600;
  background: var(--accent);
  color: #fff;
  border-radius: var(--r-md);
  box-shadow: var(--shadow-accent);
  padding: 10px 18px;
  transition: background var(--dur) var(--ease);
}
.btn-primary:hover { background: var(--accent-hover); }
```

**Правило №1: никаких сырых hex и px для брендовых величин.** Если значения нет в
токенах — сначала проверьте `colors_and_type.css`, и только в крайнем случае добавляйте
новый токен туда же (а не локальную константу).

---

## 4. Шрифты

По умолчанию грузятся с Google Fonts через `@import` в начале `colors_and_type.css`.
Все три семейства имеют полное кириллическое покрытие.

**Для продакшена** (производительность, офлайн, приватность) рекомендуется
**самостоятельный хостинг**:
1. Скачайте Golos Text (400–800), Lora (400–700 + italic), JetBrains Mono (400–600).
2. Положите в `fonts/`, опишите `@font-face`, используйте `font-display: swap`.
3. Удалите строку `@import url('https://fonts.googleapis.com/...')` из `colors_and_type.css`.

Семейства уже разведены по переменным `--font-ui`, `--font-serif`, `--font-mono` —
меняется только источник, не код компонентов.

---

## 5. Если используете Tailwind

Прокиньте CSS-переменные в тему — тогда утилиты Tailwind будут опираться на токены:

```js
// tailwind.config.js
export default {
  theme: {
    extend: {
      colors: {
        accent: 'var(--accent)',
        'accent-hover': 'var(--accent-hover)',
        paper: 'var(--paper)',
        surface: 'var(--surface)',
        ink: {
          900: 'var(--ink-900)', 700: 'var(--ink-700)',
          600: 'var(--ink-600)', 200: 'var(--ink-200)',
          // … остальные ступени из colors_and_type.css
        },
      },
      borderRadius: { md: 'var(--r-md)', lg: 'var(--r-lg)', xl: 'var(--r-xl)' },
      fontFamily: {
        ui: 'var(--font-ui)', serif: 'var(--font-serif)', mono: 'var(--font-mono)',
      },
    },
  },
};
```

Тогда `bg-accent`, `text-ink-900`, `rounded-lg`, `font-serif` ссылаются на единый
источник истины. `colors_and_type.css` всё равно подключается (он держит сами значения
переменных, шрифты и type-рампы).

## 6. Типографика на практике

Type-рампы заданы как составной токен `font:` (вес + размер/высота строки + семейство):

```css
h1.ui-title { font: var(--ui-3xl); letter-spacing: -0.02em; }   /* UI-заголовок, sans */
.score      { font: var(--mono-sm); text-transform: uppercase; } /* скоринг, mono */
.article    { font: var(--ed-body); }                            /* статья, serif */
```

Две зоны набора:
- **UI-хром** → `--ui-*` (Golos Text). Навигация, кнопки, лейблы, формы.
- **Редакционный контент** → `--ed-*` (Lora). Статьи, цитаты, крупные заголовки лендинга.
- **Системные метки** → `--mono-*` (JetBrains Mono). Имена агентов, скоринги, `HITL`.

Быстрый способ получить корректный набор без классов на каждый тег — обернуть поддерево:
- `.gr-ui` — sans-набор для интерфейса (задаёт базовый шрифт, цвет, фон, сглаживание;
  стилизует вложенные `h1/h2/h3/p`).
- `.gr-editorial` — serif-набор для статейного контента (заголовки, абзацы шириной
  `~64ch`, цитаты с красной линией-акцентом).

Это удобно для прототипа; в компонентах продакшена обычно явно ставите нужный
`font: var(--…)`.

## 7. Иконки

- Основной набор — **[Lucide](https://lucide.dev)** (скруглённый штрих ~2px). В реальном
  проекте ставьте пакет (`lucide-react`, `lucide-vue-next` и т.п.), а не CDN.
- **Каретка корректора** (знак вставки) — кастомный брендовый глиф, не из Lucide.
  Используется в логотипе, жесте «утвердить», индикаторе активной навигации, тиках
  разделов. Берите из `assets/logo-mark.svg` или инлайн-SVG (примеры — в preview-карточках
  и в `ui-kit/.../Icons.jsx`).
- **Никаких эмодзи и юникод-глифов вместо иконок.** Аватары агентов — mono-инициалы в
  тонированных чипах (`СК`, `ФЧ`), не картинки.

## 8. Motion и состояния

- Входы: `fade + подъём на 6–10px`, длительность `var(--dur)` (200ms), кривая `var(--ease)`.
  Без баунса в UI, без бесконечных декоративных циклов.
- Hover: поверхности → лёгкий well (`--ink-50/100`); акцент-кнопки темнеют до
  `--accent-hover`; карточки получают `--shadow-md` и подъём `-1px`.
- Press: ещё на ступень темнее (`--accent-active`) + `scale(0.985)`.
- Focus: `var(--ring-focus)` (синее инфо-кольцо) для инпутов; `var(--ring-accent)` для
  primary-кнопок.
- Активная навигация: `--accent-wash` pill + вертикальный красный тик-индикатор, лейбл
  → `ink-900`, вес 600.
- **Reduced motion:** всегда показывайте конечное состояние (`@media (prefers-reduced-motion: reduce)`).

## 9. Структура референс-китов (как читать разметку)

Киты — обычный React, разнесённый по JSX-файлам, которые делятся областью видимости
через `window` (так сделано, чтобы работать без сборки). При переносе вы превратите их
в нормальные модули с `import/export`.

**`ui-kit/glavred-app/`** — редакционный кабинет:
- `index.html` — монтаж, порядок загрузки (Icons и Data первыми).
- `app.css` — все стили компонентов (shell, кнопки, чипы, карточки, календарь, гейты).
- `Data.jsx` — `GR_DATA`, весь сид-контент (вынесите данные из вёрстки у себя).
- `Shell.jsx` — `Sidebar`, `Topbar`, конфиг навигации.
- `RadarPlan.jsx` — `RadarView`, `PlanView`, `InsightCard`.
- `BriefEdit.jsx` — `BriefView`, `EditView`, `Placeholder`.
- `EditorialModel.jsx` — `EditorialModelView`, `EditorialModelCard`.
- `App.jsx` — состояние, маршрутизация между разделами, тосты.

**`ui-kit/marketing-site/`** — лендинг:
- `Sections.jsx` — `Nav`, `Hero`, `Contrast`, `Newsroom`, `Workflow`, `Rubrics`,
  `FinalCTA`, `Footer`.
- `site.css` — адаптивная раскладка (одна колонка < 900px).

> Подробные карты китов — в `ui-kit/glavred-app/README.md` и
> `ui-kit/marketing-site/README.md`.

## 10. Чеклист переноса в продакшен

- [ ] `colors_and_type.css` подключён первым; все величины идут из токенов.
- [ ] Шрифты переведены на самохостинг, `@import` удалён, `font-display: swap`.
- [ ] React/Babel с CDN заменены на реальный бандлер; JSX компилируется при сборке.
- [ ] Иконки — пакет Lucide, не CDN; каретка корректора — из `assets/`.
- [ ] Данные отделены от вёрстки (по образцу `Data.jsx`).
- [ ] Состояния hover/press/focus и reduced-motion реализованы (§8).
- [ ] Русские тексты в редакторском тоне, sentence case, без эмодзи и КАПСА.
- [ ] HITL-гейты использованы там, где нужно решение человека.
- [ ] Доступность: контраст AA (токены уже подобраны под это), focus-кольца видимы,
      у иконок есть текстовые лейблы.
